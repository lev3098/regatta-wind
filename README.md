# ⛵ regatta-wind

Высокоразрешающий прогноз ветра для регат на заливе Петра Великого — локальный
**WRF 1–3 км** с учётом рельефа и берега, **сплошной сглаженный градиент скорости как у Windy**,
**коррекция поля по факту** (OpenWeatherMap + Open-Meteo), почасовые графики — всё в браузере.
Область расчёта задаётся **угловыми точками** (мыс Песчаный, б. Миноносок, о-ва Попова и
Аскольд, Кирпичный завод) — дальше прогноз не считается; ни прямоугольника, ни знаков.

## Что это

Инструмент тактической подготовки к регате на заливе Петра Великого (Владивосток).
Два движка прогноза за единым интерфейсом:

- **WRF (основной)** — настоящая мезомасштабная модель, считается **локально**. Разрешает
  береговую линию, ветровые тени за мысами/островами и **морской бриз** на 3 км (или 1 км) —
  то, что глобальные 5-км модели не видят. Рельеф учитывается нативно (WPS geogrid, SRTM/USGS).
- **Open-Meteo (фолбэк)** — грубые ~5 км онлайн, с явной пометкой, **пока WRF не посчитан**.

Оба движка отдают единое ветровое поле — карта/графики/CLI одинаковы для обоих.

### Почему именно так

Готовой бесплатной NWP-модели 1–3 км для залива Петра Великого нет (проверено: KMA/GEM не
покрывают регион, лучшее нативно — JMA MSM 5 км). Честную физику побережья надо **посчитать
локально** — этим занимается WRF. GPU при этом не используется (боевой WRF считает на CPU).

> **Платформы.** Сборка/прогон проверены **нативно на Apple Silicon (M3)** — рекомендуемый путь
> на маках (9→3 км / 12 ч ≈ ~10–20 мин). На Windows — через Docker. Детали — в
> [`wrf/README.md`](wrf/README.md).

## Установка приложения

```bash
git clone https://github.com/LeoLev/regatta-wind.git
cd regatta-wind
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Зависимости из PyPI. Ключ OpenWeatherMap уже зашит (бесплатный, без персональных
данных) — реальные данные на точках работают сразу. Свой ключ: переменная
окружения `OPENWEATHERMAP_API_KEY` (см. `.env.example`).

## Запуск

```bash
cd regatta-wind
./run.sh                       # удобный запуск (venv + streamlit)
```

Альтернативно вручную:

```bash
.venv/bin/streamlit run app.py            # без активации venv
# или, если venv активирован (source .venv/bin/activate):
streamlit run app.py
```

**Открыть в браузере:**
- на этом же компьютере → **http://localhost:8501**
- с телефона/другого ноутбука в той же сети → `http://<IP-компа>:8501`
  (узнать IP на macOS: `ipconfig getifaddr en0`; например `http://192.168.1.116:8501`).

Остановить сервер — `Ctrl+C` в терминале. Пока WRF не посчитан — показывается помеченный
фолбэк Open-Meteo (коррекция по факту работает и для него). Консоль:

```bash
python -m regatta_wind --route config/route.yaml      # auto: WRF, иначе фолбэк
python -m regatta_wind --source open-meteo            # принудительно фолбэк
```

## Локальный прогноз WRF

### macOS (Apple Silicon) — нативно

```bash
cd wrf
./build_macos.sh            # разовая сборка WRF+WPS+geog (~30–50 мин)
./download_gfs.sh           # граничные данные GFS
./run_forecast_macos.sh     # 9→3 км; MAX_DOM=3 ./run_forecast_macos.sh — добавить 1 км
```

Либо, **не выходя из приложения**, в боковой панели выбери точность (**3 км** — быстрее,
**1 км** — точнее) и горизонт (на сколько часов вперёд), затем нажми **«🧮 Просчитать …»** —
WRF посчитается в фоне на залив, со шкалой прогресса; по готовности карта обновится сама.
Прогноз показывается **от текущего момента** до выбранного времени (прошлые часы не рисуются).

### Windows — Docker

См. [`wrf/README.md`](wrf/README.md): `docker compose run --rm wrf`.

`wrfout_d02_*` ложится в `wrf/output/` — приложение подхватывает его автоматически.

## Интерфейс

**🗺 Карта ветра** — сплошной **сглаженный градиент скорости** (как Windy, без «квадратов»
сетки), стрелки направления (плотность регулируется), цветовая шкала. Белая рамка — **граница
области расчёта**, подписи по углам — именованные пределы. Оранжевые точки — **реальный ветер
OWM** по сетке акватории. Ползунок времени — с текущего часа. Наведи на поле — скорость / напр. / порыв.

**📈 Графики по точкам** — по каждой угловой точке: скорость (+порывы), направление с заходами
← →, тактическое окно, амплитуда и характер ветра.

**Коррекция по факту** — поле сдвигается к реальным наблюдениям (OWM + Open-Meteo current,
сетка по акватории) на ближних часах, затухая к концу горизонта. Тумблер в боковой панели.

**Боковая панель**
- источник (Авто / WRF / фолбэк);
- **точность 3 км / 1 км** и **горизонт** (часов вперёд) → **кнопка расчёта со шкалой прогресса**;
- **коррекция по факту** (вкл/выкл);
- вид: макс. шкала (узлы), непрозрачность поля, **плотность стрелок**.

## Настройка

`config/route.yaml`:

```yaml
forecast:
  source: wrf            # wrf | open-meteo
  hours_ahead: 12        # горизонт фолбэка/OWM (в WRF выбирается кнопкой)
  tactical_window: [3, 5]
  fallback_model: jma_seamless
wrf:
  output_dir: wrf/output
  domain: d02            # d02 = 3 км, d03 = 1 км
area:                    # центр считается по угловым точкам; half_span — рамка карты
  half_span_deg: 0.50
landmarks:               # угловые точки = ГРАНИЦЫ области расчёта (не точки OWM)
  - {name: Мыс Песчаный, lat: 43.34, lon: 131.79}
  - {name: Остров Аскольд, lat: 42.78, lon: 132.34}
```

Координаты угловых точек приблизительные — правь прямо в `landmarks:`. Свой ключ OWM —
в `.env` (см. `.env.example`).

## Ограничения

- **WRF — не магия.** 3 км ловит бриз и геометрию берега, но это даунскейлинг GFS без локальной
  ассимиляции. Сверяй с тем, что видишь на воде.
- **Считается не мгновенно.** На M3 — минуты, на старом CPU — десятки минут/ночь. GPU не помогает.
- **Фолбэк грубый.** ~5 км без рельефа; для гонки опирайся на WRF-поле.
- Время WRF в UTC, в интерфейсе переведено в `Asia/Vladivostok`.
