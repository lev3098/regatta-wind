# ⛵ regatta-wind

Оперативный прогноз ветра для маршрутных гонок — интерактивная карта и почасовые таблицы прямо в браузере.

## Что это

Инструмент для тактической подготовки к регате на заливе Петра Великого (Владивосток). Запрашивает прогноз ветра по сетке точек через [Open-Meteo API](https://open-meteo.com/en/docs/jma-api) и отображает его двумя способами:

**🗺 Карта ветра** — стрелки направления поверх цветового градиента скорости. Ползунок времени — скрол по часам прогноза. Рамку прогнозируемой площади можно перемещать.

**📈 Графики** — почасовые кривые скорости и направления по каждой точке дистанции: тактическое окно подсвечено, заходы ветра отмечены стрелками.

### Источник данных

Модель **JMA MSM** — японская мезомасштабная модель, 5 км разрешение, 4D-Var ассимиляция, обновление каждые **3 часа**. Целиком покрывает Японское море и залив Петра Великого. Доступ бесплатный, без ключа, лицензия данных CC BY 4.0.

## Скриншоты

### Карта ветра с оверлеем скорости
Стрелки показывают куда идёт ветер, длина пропорциональна скорости.  
Цветовой градиент (синий → жёлтый → красный) — скорость ветра по всей акватории.

```
 0–5 kn   ■ синий
 5–10 kn  ■ голубой
10–15 kn  ■ зелёный
15–20 kn  ■ жёлтый
20–25 kn  ■ оранжевый
  25+ kn  ■ красный
```

## Установка

```bash
git clone https://github.com/your-username/regatta-wind.git
cd regatta-wind

python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

Зависимостей немного, всё из PyPI, ключ API не нужен.

## Запуск

### Веб-интерфейс (рекомендуется)

```bash
streamlit run app.py
```

Откроется браузер на `http://localhost:8501`.

### Консольный режим

```bash
python -m regatta_wind --route config/route.yaml
```

Выводит таблицу прямо в терминал через `rich`.

## Настройка маршрута

Отредактируй `config/route.yaml` — впиши реальные координаты своих знаков:

```yaml
name: Залив Петра Великого — гоночная дистанция
timezone: Asia/Vladivostok

forecast:
  model: jma_seamless
  wind_speed_unit: kn
  hours_ahead: 24
  past_hours: 3
  tactical_window: [3, 5]   # окно принятия решений, ч от текущего момента

waypoints:
  - name: Старт
    lat: 43.115
    lon: 131.870
  - name: Знак 1
    lat: 43.090
    lon: 131.920
  - name: Знак 2
    lat: 43.060
    lon: 131.880
  - name: Финиш
    lat: 43.115
    lon: 131.870
```

Все точки запрашиваются **одним** обращением к API.

## Как читать карту

| Элемент | Описание |
|---|---|
| **Стрелки** | Направление движения воздуха (куда дует). Длина ∝ скорости |
| **Цветовой оверлей** | Скорость ветра по площади (синий = тихо, красный = сильно) |
| **Ползунок времени** | Скрол по часам прогноза. Серые часы — модельное прошлое |
| **Рамка** | Площадь, по которой запрошен прогноз. Позицию можно двигать |
| **Белые маркеры** | Знаки гоночной дистанции из `route.yaml` |

## Как читать графики

| Элемент | Описание |
|---|---|
| **Синяя линия** | Скорость ветра, kn |
| **Жёлтая зона** | Тактическое окно 3–5 ч — время принятия ключевых решений |
| **Вертикальная черта** | Текущий момент |
| **← →** на графике направления | Заходы ≥15°: ← влево (back), → вправо (veer) |
| **Амплитуда** | Размах колебаний направления. >30° = осциллирующий ветер, ходи на заходах |

## Ограничения модели

- **5 км** — хорошо ловит синоптику и общий бриз, но не разрешает узкие бризовые фронты и обтекание мысов. Расстояние от знака до узла сетки показано в графиках — если велико, относись к значению осторожно.
- Модель не обновляется в реальном времени: новый ранс MSM выходит каждые **3 часа** — жми «Обновить прогноз» при появлении связи.
- Данные прошлого (`past_hours`) — сверяй с тем, что видишь на воде: это лучший способ понять, насколько модель врёт в конкретный день.
